- RAG(Retrieval Augmented Generation)

  - 프라이빗 또는 독점적 데이터로 텍스트 생성을 보완하는 기술이다.
    - 대규모 데이터셋 또는 지식 기반(knowledge base)을 검색하기 위해 설계된 모델과 해당 모델을 통해 검색된 정보를 받아와 text 응답을 생성하는 대규모 언어 모델(LLM)과 같은 생성 모델을 결합한다.
    - 이를 통해 모델을 다시 훈련할 필요 없이 대규모 언어 모델의 출력을 향상시킬 수 있다. 
    - 지식 기반은 사람이나 시스템이 참고할 수 있도록 정리, 저장된 정보의 집합을 의미한다.
    - LLM 맥락에서 지식 기반은 주로 LLM이 직접 학습하지 않았거나, 최신이거나, 특정 도메인에 특화된 정보를 담고 있는 외부 저장소를 의미한다.
  - RAG의 동작 방식은 아래와 같다.
    - 사용자가 질의를 입력하고, 이 질의는 vector로 변환된다.
    - Embedding된 사용자의 질의와 가장 유사한 vector들을 조회한다.
    - 검색된 문서를 프롬프트에 포함하여 LLM에 전달한다.
    - LLM이 검색 결과를 근거로 응답을 생성한다.
  - Prompting
    - 생성형 AI에 input을 전달하는 과정이다.
    - Model의 output은 model에 얼마나 input을 잘 전달했는지에 따라 달라진다.
    - 이러한 작업을 prompt engineering이라고 부른다.
    - 더 자세한 설명은 [링크](https://www.promptingguide.ai/kr) 참조

  - Fine-tuning 방식
    - 이미 학습된 LLM의 가중치를 추가 데이터 학습을 통해 미세하게 조정하여 응답을 바꾸는 과정이다.
    - 지식을 조회하는 과정이 아니라 모델 내부 계산 결과의 확률 분포를 바꾸는 과정이다.
    - RAG와의 차이는 RAG는 검색한 결과를 prompt를 통해 LLM에 추가적인 지식을 주입하여 그 지식을 근거로 응답을 생성하는 방식인 반면, fine-tuning은 추가 지식을 LLM이 학습하여 계산 결과의 확률 분포를 바꾸는 방식이라는 것이다.
    - 즉, RAG의 경우 검색이 실행 되지만, fine-tuning의 경우 (결과적으로 검색에 사용되기는 하지만) 엄밀히 말해 검색은 아니다.
    - RAG는 광범위한 외부 지식을 검색하여 이를 텍스트 생성과 결합하는 방식인 반면, fine-tuning은 특정 목적을 위해 비교적 제한된 범위의 데이터에 집중한다.
    - 외부 자원에 접근할 수 있다는 점에서, RAG는 웹이나 기업 내부 지식 베이스로부터 실시간 또는 동적으로 변하는 정보를 활용해 응답을 생성해야 하는 작업에 특히 유용하다. 
    - 반면 fine-tuning은 수행해야 할 작업이 명확하게 정의되어 있고, 그 작업 하나에 대한 성능 최적화가 목표라면 fine-tuning은 매우 효율적인 방법이 될 수 있다. 
    - 두 기법 모두 각 작업마다 LLM을 처음부터 다시 학습시킬 필요가 없다는 공통적인 장점을 가진다.

  - RAG의 장점
    - Retrieval Augmented Generation(RAG)은 단독으로 동작하는 언어 모델에 비해 여러 가지 장점을 가진다.
    - RAG는 외부 참조 데이터를 정기적으로 갱신할 수 있기 때문에, 모델이 최신의 사실과 관련 정보를 항상 접근할 수 있도록 보장한다. 이를 통해 사용자의 질의와 관련될 수 있는 최신 정보를 반영한 응답을 생성할 수 있다.
    - RAG는 더 적은 연산 자원과 저장 공간을 필요로 하기 때문에 비용 효율적인 선택지이다. 자체적으로 LLM을 보유하거나, 모델을 fine-tuning하는 데 시간과 비용을 들일 필요가 없다.
    - RAG는 응답의 근거가 되는 외부 출처를 인용하여 사용자에게 제공할 수 있다. 사용자는 필요에 따라 해당 출처를 직접 확인함으로써, 자신이 받은 응답이 정확한지 검증할 수 있다.
    - LLM 기반 챗봇은 과거의 정형화된 응답 방식보다 개인화된 답변을 생성할 수 있지만, RAG는 그보다 한 단계 더 나아간 맞춤형 응답을 제공할 수 있다. 이는 RAG가 의도를 판단하는 과정에서 검색 기반 기법(주로 시맨틱 검색)을 활용하여, 문맥에 기반한 다양한 정보를 참조하면서 응답을 종합할 수 있기 때문이다.
    - 학습되지 않은 복잡한 질의에 직면했을 때, LLM은 때때로 부정확한 답변을 생성하는 ‘환각(hallucination)’ 현상을 보일 수 있지만 RAG는 관련 데이터 소스로부터 추가적인 참조 정보를 제공하여 응답을 근거에 기반해 생성함으로써, 모호한 질문에도 보다 정확하게 대응할 수 있다.
    - RAG 모델은 활용 범위가 넓어, 대화 시스템, 콘텐츠 생성, 정보 검색 등 다양한 자연어 처리 작업에 적용할 수 있다.
    - RAG는 검증된 외부 출처에 의존함으로써, 응답에 포함될 수 있는 편향(bias)을 완화하는 데 도움을 줄 수 있다.
  - RAG의 한계
    - RAG는 외부 지식에 의존하므로 검색된 정보가 부정확할 경우, 그에 기반해 생성되는 결과 역시 부정확해질 수 있다.
    - RAG의 검색(retrieval) 구성 요소는 대규모 지식 베이스나 웹을 대상으로 탐색을 수행하므로, 계산 비용이 크고 처리 속도가 느릴 수 있다. 다만 이는 여전히 fine-tuning에 비해서는 더 빠르고 비용도 적게 든다.
    - 검색과 생성 구성 요소를 매끄럽게 통합하기 위해서는 세심한 설계와 최적화가 필요하며, 이 과정에서 학습과 배포 단계 모두에서 잠재적인 어려움이 발생할 수 있다.
    - 외부 소스로부터 정보를 조회하는 과정은 민감한 데이터를 다룰 경우 개인정보 보호 이슈를 야기할 수 있다. 또한 개인정보 보호 및 컴플라이언스 요구 사항을 준수해야 하기 때문에, RAG가 접근할 수 있는 정보 소스가 제한될 수도 있다. 다만 문서 단위 접근 제어(document-level access)를 적용하여, 특정 역할(role)에 대해서만 접근 및 보안 권한을 부여하는 방식으로 이러한 문제를 해결할 수 있다.
    - RAG는 사실적 정확성에 기반한 접근 방식이기 때문에, 상상력이나 허구적 요소가 필요한 콘텐츠 생성에는 어려움을 겪을 수 있으며, 이로 인해 창작형 콘텐츠 생성 용도로의 활용에는 한계가 있다.
